# 骨架模板 — Skill 构建流程通用框架

本文件定义生成 SKILL.md 时"构建流程"章节的通用框架。
Compile 命令在组装 Step 3.8 时，用知识文档的领域内容填充此框架。

---

## 通用构建流程

```
┌─────────────────────────────────────────────────┐
│ 步骤 0: 语义解析                                  │
│   - 识别用户意图中的核心动作                        │
│   - 按语义分解维度拆解请求                          │
│   - 逐条检查消歧规则，消除歧义                      │
│   - 确定需要调用的工具集合                          │
├─────────────────────────────────────────────────┤
│ 步骤 1: 数据检索                                  │
│   - 确认所需工具及调用顺序                          │
│   - 依次调用工具获取数据                            │
│   - 解析返回结果，提取后续步骤所需数据                │
│   - 如有依赖关系，先调用上游工具                     │
├─────────────────────────────────────────────────┤
│ 步骤 2: 参数构造                                  │
│   - 逐参数填充，不遗漏                             │
│   - 参考 Quick Reference 确定参数值                │
│   - 应用 Filter Routing 选择筛选路径               │
│   - 检查 Composition Constraints 防止非法组合      │
│   - 应用默认策略填充用户未指定的参数                  │
├─────────────────────────────────────────────────┤
│ 步骤 3: 自检清单                                  │
│   - 每条 Iron Rule → 一个检查项                    │
│   - 通用检查：返回解析正确、格式正确、输出规范         │
│   - 逐项过一遍，全部通过才提交请求                   │
├─────────────────────────────────────────────────┤
│ 步骤 4: 执行与格式化输出                           │
│   - 提交请求                                      │
│   - 按输出展示规范格式化结果                         │
│   - 如有异常 → 参考错误模式诊断                     │
└─────────────────────────────────────────────────┘
```

## 填充指南

### 步骤 0 占位符

| 占位符 | 数据来源（知识文档） |
|--------|-------------------|
| `{分解维度列表}` | `## Semantic Framework` → Decomposition Dimensions |
| `{消歧规则列表}` | `## Semantic Framework` → Disambiguation Rules |
| `{工具集合}` | `## Tool Interface` → 工具名列表 |

### 步骤 1 占位符

| 占位符 | 数据来源（知识文档） |
|--------|-------------------|
| `{工具调用指令}` | 每个工具一条："调用 {tool} 获取 {purpose}" |
| `{调用顺序}` | `## Tool Interface` → 如有依赖关系 |
| `{返回解析}` | `## Tool Interface` → 返回格式与解析规则 |

### 步骤 2 占位符

| 占位符 | 数据来源（知识文档） |
|--------|-------------------|
| `{参数列表}` | `## Request Specification` → Parameters |
| `{Quick Reference}` | `## Request Specification` → Quick Reference |
| `{Filter Routing}` | `## Request Specification` → Filter Routing |
| `{Composition Constraints}` | `## Request Specification` → Composition Constraints |
| `{Default Strategies}` | `## Request Specification` → Default Strategies |

### 步骤 3 填充规则

```
对于 Iron Rules 中的每一条规则:
  生成: "✅ {将规则标题改写为检查项}"

示例:
  规则: "日期范围必须使用 ISO 8601 格式"
  →  检查项: "✅ 日期参数使用 ISO 8601 格式 (YYYY-MM-DD)"

  规则: "粒度参数必须全小写"
  →  检查项: "✅ 粒度参数全小写 (daily, weekly, monthly)"
```

追加通用检查项（始终包含）：
```
✅ 工具返回的数据已正确解析
✅ 请求格式正确（转义、类型、必填项）
✅ 输出结构符合展示规范
```
